Part 1 — Audit Summary
Five structural issues were identified in the original MBS Question Flow. All are resolved in the final integrated flow below.
1.1  Duplicates & Redundancies Removed (18 questions)
The following 18 questions were flagged for removal due to exact or near-identical wording measuring the same construct:
3A / 3H:  Leader or follower (Phase 3A) vs. "I prefer leading" (Phase 3H) — identical construct, keep 3A
3B / 3K:  Masculine vs. feminine slider duplicated in Phase 3B and 3K
3F / 3P:  Both probe creative vs. analytical preference with near-identical wording
3N / 3R:  Both measure preference for routine vs. variety
3S / 3E:  Social comfort questions with trivially different framing
3D / 3X:  Risk tolerance measured twice with different labels
4C-4 / 4C-8 / 4C-10:  Three overlapping political ideology items; retain 4C-4 only
4A-2 / 4A-3:  Both measure financial ambition; 4A-2 is more nuanced, remove 4A-3
4B-1 / 4C-1 / 4C-5:  Leadership aspiration measured 3 times across phases 4B and 4C
1.2  Overdone Dimensions (4–8 items for same construct)
Individuality / Introversion-Extraversion:  8 items → trimmed to 5 (1A-1, 1A-2, 1A-3, 1A-5, 2D-2)
Political Ideology:  7 items → trimmed to 4 (retain items with highest construct validity)
Leadership Aspiration:  4 near-duplicates across phases 3, 4A, 4B → keep best 2
Money vs. Recognition motivation:  3 near-identical items → keep 1C-1 only
1.3  Chronology Issues Fixed
Real Problems moved Phase 5 → Session 3:  Students need cognitive baseline before open problem reflection is meaningful
Avoidances moved Phase 1 → Session 3:  Avoidance framing creates negative priming at start; better after personality established
Subjects moved Phase 2 → Session 3 Phase 11:  Domain knowledge games now replace static subject checklist
Aptitude/Knowledge moved earlier → Session 2:  Cognitive games now run in dedicated session, not buried in Phase 6
AI interview now continuous, not end-of-flow only:  AI follow-ups integrated at each phase rather than a single Phase 7 block

Phase 6  —  Extended Cognitive Battery  |  M9–M13: working memory, attention, flexibility
M9  |  Dual N-Back Lite  |  ~4 min  |  Session 2
Measures: Working Memory Capacity, Divided Attention
Mechanic: Two simultaneous streams: a letter spoken aloud and a grid position lit up. Student taps when either matches what appeared 2 steps back. N increases adaptively from 1-back to 3-back based on accuracy.
O*NET Dimensions: O*NET Abilities: Working Memory Capacity; also signals: Auditory Attention, Selective Attention
Tech Stack: Phaser.js + Web Audio API
Items / Trials: ~40 trials (adaptive)
M10  |  Split Track  |  ~4 min  |  Session 2
Measures: Time Sharing, Divided Attention, Multitasking ability
Mechanic: Two simultaneous tasks on screen: sort falling objects by colour on the left, respond to number parity (odd/even tap) on the right. Scoring: dual-task cost = performance on each task alone vs. together.
O*NET Dimensions: O*NET Abilities: Time Sharing; O*NET Skills: Time Management proxy
Tech Stack: Phaser.js (two independent Phaser groups running simultaneously)
Items / Trials: 30 dual-task trials
M11  |  Go/No-Go  |  ~3 min  |  Session 2
Measures: Impulse Control, Sustained Attention, Response Inhibition
Mechanic: Shapes appear rapidly. Tap for circles (Go), do NOT tap for red circles (No-Go). Speed and error rate scored. Later rounds introduce distractors (flashing borders, similar shapes). False alarms (tapping No-Go) penalise more than misses.
O*NET Dimensions: O*NET Abilities: Selective Attention; O*NET Work Styles: Self-Control, Cautiousness
Tech Stack: Phaser.js (fast animation loop, touch + keyboard)
Items / Trials: 60 trials (40 Go, 20 No-Go)
M12  |  Colour-Word Stroop  |  ~4 min  |  Session 2
Measures: Cognitive Flexibility, Inhibitory Control, Processing Speed
Mechanic: The word "RED" appears in blue ink. Student types the INK COLOUR, not the word meaning. Four conditions in order: (1) colour patches only, (2) congruent words, (3) incongruent words, (4) mixed. Stroop interference score = RT difference between condition 2 and condition 3.
O*NET Dimensions: O*NET Abilities: Selective Attention, Perceptual Speed; O*NET Work Styles: Self-Control, Adaptability
Tech Stack: Phaser.js (text rendering + response time logging in milliseconds)
Items / Trials: 50 trials across 4 conditions
M13  |  Rule Shift  |  ~5 min  |  Session 2
Measures: Category Flexibility, Adaptability to Change, Cognitive Set-Shifting
Mechanic: Wisconsin Card Sort inspired. Cards appear with attributes (colour, shape, number). Student sorts by one rule. Without warning, the sorting rule changes. Student must detect the change and shift strategy. Two rule changes total.
O*NET Dimensions: O*NET Abilities: Category Flexibility; O*NET Work Styles: Adaptability, Tolerance for Ambiguity
Tech Stack: Phaser.js (card sorting with silent rule-change events)
Items / Trials: ~40 trials with 2 rule shifts
Phase 7  —  Memory Systems  |  M14–M15: associative memory and information ordering
M14  |  Name-Face-Role  |  ~5 min  |  Session 2
Measures: Associative Memory, Memory for Names and Faces
Mechanic: 8 "people" shown for 5 seconds each — name, face (generated), and job title. Then tested in random order: show face → type name, show name → select role from 4 options. Relevant for social, HR, sales, consulting, teaching careers.
O*NET Dimensions: O*NET Abilities: Memorisation, Associative Memory; O*NET Knowledge: Customer and Personal Service
Tech Stack: Phaser.js (face card display + recall interface)
Items / Trials: 16 recall items (8 name + 8 role)
M15  |  Story Sequence  |  ~4 min  |  Session 2
Measures: Information Ordering, Sequential Reasoning, Process Understanding
Mechanic: A 6-step real-world process shown in jumbled order (scientific experiment, patient intake, construction sequence, event planning — 3 different domain-neutral scenarios). Student drags steps into correct order. Scored on number of correct adjacencies, not just total correctness.
O*NET Dimensions: O*NET Abilities: Information Ordering; O*NET Skills: Active Learning
Tech Stack: Phaser.js (drag-and-drop sequence builder)
Items / Trials: 3 sequences × 6 steps = 18 ordering decisions
Phase 8  —  Social Cognition & Creativity  |  M16–M19: reading people, divergent thinking
M16  |  Micro-Read  |  ~5 min  |  Session 2
Measures: Social Perceptiveness, Emotion Recognition, Non-Verbal Sensitivity
Mechanic: A face appears for 200–500ms, student selects emotion from 6 options (including ambiguous: contempt, pride, discomfort alongside basic 6). 20 faces total. Scored on accuracy + confidence calibration (penalise overconfident wrong answers). Critical differentiator for healthcare, counselling, teaching, HR, sales.
O*NET Dimensions: O*NET Abilities: Speech Recognition proxy; O*NET Skills: Social Perceptiveness; O*NET Work Styles: Empathy
Tech Stack: Phaser.js (rapid face display + choice buttons; face images pre-rendered)
Items / Trials: 20 face stimuli
M17  |  Conversation Fork  |  ~6 min  |  Session 2
Measures: Social Judgment, Interpersonal Skill, Tact, Service Orientation
Mechanic: Text-based branching scenarios — ambiguous workplace/social situations. Three decision points, each with 3 response options, each with visible downstream consequences shown after choice (not right/wrong — shows what actually happens next). Scores: prosocial choice rate, escalation avoidance, acknowledgment of others.
O*NET Dimensions: O*NET Skills: Social Perceptiveness, Service Orientation, Coordination; O*NET Work Styles: Cooperation, Empathy
Tech Stack: Phaser.js (text adventure engine) or HTML/CSS state machine
Items / Trials: 4 scenarios × 3 decision points = 12 choices
M18  |  Alternative Uses (AI-Scored)  |  ~5 min  |  Session 2
Measures: Originality, Divergent Thinking, Ideational Fluency, Creative Thinking
Mechanic: Show 3 everyday objects (paperclip, rubber band, shoe). Student types as many uses as possible in 90 seconds each. Claude API scores each response for: fluency (count), originality (unusual vs. common), category spread (diverse domains), elaboration (detail level). Completely impossible to cheat — every run generates unique scoring baseline.
O*NET Dimensions: O*NET Abilities: Fluency of Ideas, Originality; O*NET Work Styles: Innovation, Intellectual Curiosity
Tech Stack: HTML text input + Claude API scoring (JSON output: fluency, originality, spread, elaboration per response)
Items / Trials: 3 objects × ~10 uses avg = ~30 scored responses
M19  |  Idea Mesh  |  ~4 min  |  Session 2
Measures: Creative Synthesis, Associative Thinking, Conceptual Flexibility
Mechanic: Two unrelated concepts shown (e.g. "submarine" + "hospital", "music" + "engineering"). Student writes a connection in one sentence. 5 pairs total. Claude API scores: coherence of connection (is it logical?), creativity (is the bridge surprising?), specificity (vague vs. detailed).
O*NET Dimensions: O*NET Abilities: Originality, Fluency of Ideas; O*NET Work Styles: Intellectual Curiosity, Innovation
Tech Stack: HTML text input + Claude API scoring
Items / Trials: 5 concept pairs
Phase 9  —  Analytical & Systems Thinking  |  M20–M22: problem sensitivity, systems, fine motor
M20  |  Spot the Break  |  ~5 min  |  Session 2
Measures: Problem Sensitivity, Quality Control Analysis, Critical Evaluation
Mechanic: A process diagram or plan is shown with one or two deliberate flaws (wrong step order, missing safety check, logical impossibility). Student identifies the problem by clicking it and briefly explaining why. 5 rounds: household analogy → business process → technical system. No domain knowledge required for rounds 1–3.
O*NET Dimensions: O*NET Skills: Quality Control Analysis; O*NET Abilities: Problem Sensitivity; O*NET Work Styles: Attention to Detail
Tech Stack: Phaser.js (flowchart renderer with clickable nodes)
Items / Trials: 5 scenarios × 1–2 flaws each
M21  |  Domino Effect  |  ~5 min  |  Session 2
Measures: Systems Analysis, Causal Reasoning, Systems Thinking
Mechanic: A scenario event shown ("Factory shifts schedule 1 hour earlier"). A chain of 6 downstream effects listed — student selects which are genuine consequences and which are spurious. Three scenarios of increasing complexity (household → organisational → ecological).
O*NET Dimensions: O*NET Skills: Systems Analysis, Systems Evaluation; O*NET Work Styles: Analytical Thinking
Tech Stack: Phaser.js (cause-effect chain visual with checkbox selection)
Items / Trials: 3 scenarios × 6 effects = 18 causal judgments
M22  |  Steady Trace  |  ~4 min  |  Session 2
Measures: Fine Motor Control, Eye-Hand Coordination, Psychomotor Precision
Mechanic: A path appears on screen (varying complexity — straight, curved, maze-like). Student traces it with mouse/trackpad as precisely as possible under time pressure. 8 paths of increasing complexity. Scored: deviation from path (accuracy) + time to complete (speed). First ability test for fine motor control possible without physical apparatus.
O*NET Dimensions: O*NET Abilities: Wrist-Finger Speed, Arm-Hand Steadiness, Speed of Limb Movement; relevant for surgery, dentistry, architecture, craft
Tech Stack: Phaser.js (pointer tracking, deviation from SVG path calculated in real time)
Items / Trials: 8 trace paths

Phase 10  —  Technology & ICT Skills Assessment  |  T1–T7: game mechanics embed the skill measurement

T1  |  Logic Flow  |  ~8 min  |  Session 3
Measures: Programming Logic, Technology Design, Computational Thinking
Mechanic: Visual block-programming interface (no syntax). Student arranges logic blocks (move, loop, if-condition) to make a character reach a goal. 5 levels: (1) linear sequence, (2) loops, (3) conditionals, (4) nested logic, (5) debugging a broken program. The process reveals more than the answer — loop vs. brute-force reveals computational elegance.
O*NET Dimensions: O*NET Skills: Programming, Technology Design, Operations Analysis; O*NET Knowledge: Computers and Electronics
Tech Stack: Phaser.js with Blockly-style drag-and-drop block system (open source: Google Blockly)
Items / Trials: 5 levels, adaptive difficulty
T2  |  Debug Den  |  ~6 min  |  Session 3
Measures: Troubleshooting, Quality Control Analysis, Systems Diagnosis
Mechanic: A visual process diagram or pseudocode snippet has 1–2 deliberate errors per round. Student identifies and fixes them under time pressure. 5 rounds: (1) household analogy (recipe with wrong step), (2) business process, (3) data flow diagram, (4) algorithm pseudocode, (5) server request flowchart. Rounds 1–2 accessible without tech background.
O*NET Dimensions: O*NET Skills: Troubleshooting, Quality Control Analysis, Operations Analysis; O*NET Knowledge: Computers and Electronics, Engineering and Technology
Tech Stack: Phaser.js (interactive flowchart with clickable error zones)
Items / Trials: 5 rounds, 1–2 bugs each
T3  |  Network Weave  |  ~6 min  |  Session 3
Measures: Systems Analysis, Network Architecture, Telecommunications Reasoning
Mechanic: Data packets travel from source to destination across a network of nodes. Student routes them in real time — drag connections, avoid congested nodes, prioritise urgent (red) packets. 4 scenarios of increasing network complexity. Phaser A* pathfinding used internally; student makes routing decisions not code.
O*NET Dimensions: O*NET Skills: Systems Analysis, Technology Design; O*NET Knowledge: Telecommunications, Computers and Electronics
Tech Stack: Phaser.js (real-time routing game, Matter.js for visual physics)
Items / Trials: 4 network scenarios
T4  |  Spreadsheet Sprint  |  ~6 min  |  Session 3
Measures: Operations Monitoring, Data Literacy, Quantitative Analysis
Mechanic: Business scenarios with embedded data table (3 columns, 8 rows). Student applies filters/sorting to answer a specific question: "Which product will run out first?", "Which salesperson had the best month?". 4 tasks. Not a full Excel simulation — focused on data reasoning.
O*NET Dimensions: O*NET Skills: Operations Monitoring, Quality Control Analysis; O*NET Knowledge: Computers and Electronics, Economics and Accounting
Tech Stack: HTML table with interactive filter/sort controls (no spreadsheet library needed)
Items / Trials: 4 data tasks
T5  |  Phishing Patrol  |  ~5 min  |  Session 3
Measures: ICT Security Awareness, Critical Evaluation, Attention to Detail
Mechanic: 6 emails/messages shown one at a time. Student flags legitimate vs. suspicious. Scoring: false positives and false negatives scored asymmetrically (missing a real threat costs more). Later rounds include social engineering (urgency manipulation, impersonation). Feels like a real-world challenge.
O*NET Dimensions: O*NET Skills: Quality Control Analysis; O*NET Knowledge: Computers and Electronics; O*NET Work Styles: Attention to Detail
Tech Stack: Phaser.js or pure HTML/CSS (email mockup display)
Items / Trials: 6 messages, 3 rounds of increasing subtlety
T6  |  API Mapper  |  ~7 min  |  Session 3
Measures: Technology Design, Systems Integration, Data Architecture
Mechanic: Visual node-graph interface. Left panel: data sources (weather feed, sales DB, calendar). Right panel: output widgets (report, alert, chart). Centre: logic nodes (filter, aggregate, transform). Student connects them correctly for 4 scenarios of increasing complexity. Tests integration thinking without requiring code knowledge.
O*NET Dimensions: O*NET Skills: Technology Design, Programming; O*NET Knowledge: Computers and Electronics, Engineering and Technology
Tech Stack: Phaser.js (custom node-graph with line-drawing between draggable nodes)
Items / Trials: 4 integration scenarios
T7  |  Equipment Selector  |  ~5 min  |  Session 3
Measures: Operations Analysis, Equipment Selection, Constraint Satisfaction
Mechanic: Given a job requirement (build a 3m wall in 6 hours, budget 500 units) and a catalogue of tools with specs, student selects the best combination. 4 problems: (1) construction equipment, (2) lab instruments, (3) computing hardware selection, (4) emergency equipment allocation. Pure constraint satisfaction — no domain jargon in rounds 1–2.
O*NET Dimensions: O*NET Skills: Operations Analysis, Equipment Selection, Installation; O*NET Knowledge: Engineering and Technology, Building and Construction
Tech Stack: HTML (filterable equipment catalogue with running cost/time counter)
Items / Trials: 4 selection problems
Phase 11  —  Knowledge Domain Games  |  K1–K10: contextual not recall-based, adaptive selection
Each student receives 3–5 knowledge games matched to their career domain shortlist from Sessions 1–2. All 10 are described here; only the relevant subset is deployed per student. M7 Expert Challenge runs after the domain games to probe knowledge depth adaptively.
K1  |  World Map Mind  |  ~6 min  |  Session 3
Measures: Geographic Knowledge, Spatial Reasoning, Cultural Literacy
Mechanic: Clues appear one at a time — climate type, landmark description, cultural fact, economic indicator. Student narrows down to a location by clicking progressively smaller map regions. More clues used = lower score. 5 puzzles of increasing geographic specificity. GeoGuessr-inspired but scored on inference reasoning not guessing.
